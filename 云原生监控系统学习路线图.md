# 云原生监控学习路线图（可打印）

---

## 1. 基础与概念

* 监控核心三要素：Metrics / Logs / Traces
* 可观测性 vs 监控（Observability vs Monitoring）
* Exporter、Agent、SDK 的区别
* Pull vs Push 模型
* 时序数据库（TSDB）基础

---

## 2. Prometheus 基础

* Prometheus 架构（Scrape、TSDB、Rule、Alert）
* /metrics 暴露格式
* PromQL 基础查询
* 指标类型（Counter / Gauge / Histogram / Summary）
* Service Discovery（K8s、Consul、Static）

---

## 3. PromQL 进阶

* Rate、Increase、Offset
* 向量匹配：on / ignoring / group_left
* 子查询：`metric[5m:1m]`
* Recording Rules（记录规则）
* Alerting Rules（告警规则）

---

## 4. Exporters

* Node Exporter
* Kubernetes Metrics
* cAdvisor
* Blackbox Exporter
* Custom Exporter 编写

---

## 5. 可观测性数据管道

* Remote Write / Remote Read
* Prometheus Sharding
* Metrics Gateway（例如 VictoriaMetrics Gateway）
* 高基数处理策略

---

## 6. Alertmanager

* 分组（Grouping）
* 去重（Deduplication）
* 抑制（Inhibition）
* 静默（Silence）
* 多实例 HA
* 告警路由

---

## 7. Grafana 可视化

* Dashboard 结构
* Variables（模板变量）
* Alerting（Grafana 原生告警）
* 面板类型选择
* 云原生可视化最佳实践

---

## 8. Kubernetes 监控体系

* kube-state-metrics
* Metrics Server vs Prometheus
* Pod / Node / Deployment 指标体系
* 资源利用率监控

---

## 9. Thanos / Cortex / Mimir

* Sidecar
* Store
* Ruler
* Query / Query Frontend
* 对象存储（S3 / GCS / MinIO）
* 多集群统一视图

## 10. 日志 Observability

* Loki 架构
* LogQL 查询
* Promtail Agent
* 日志标签设计

---

## 11. 链路追踪

* OpenTelemetry（OTel）概念
* Trace → Metrics → Logs 互相关联
* Tempo / Jaeger 基础
* 上下游关联分析

---

## 12. 整体可观测平台

* 云原生全栈架构：Metrics / Logs / Traces
* 单体 Prometheus → 分布式 → SaaS
* 高可用、持久化、水平扩展
* 性能优化与成本优化

## 13. 实战项目（建议路线）

1. 搭建单机 Prometheus + Grafana
2. 加入 Node Exporter + kube metrics
3. 写 10 条 PromQL
4. 配置 5 条告警 + Alertmanager 告警路由
5. 部署 Thanos 打通长时存储
6. 部署 Loki + Promtail
7. 部署 Tempo + OTel Collector
8. 打通 Metrics + Logs + Traces 全链路
